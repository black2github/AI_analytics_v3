# Единая таблица полей экранной формы

Все поля формы — в **одной** таблице (не дробить по подразделам). Порядок колонок
фиксирован (обязательность — перед логикой заполнения):

```
| Поле | Тип | Формат | Представление | Обяз. | Логика заполнения |
```

## Правила заполнения колонок

- **Поле** — имя поля как на форме.
- **Тип** — тип элемента данных из словаря типов, если поле один-к-одному
  соответствует атрибуту; `—` если общего типа нет (фронтовое поле, расходится с
  атрибутом).
- **Формат** — ограничение допустимых значений (категория A) с пометкой источника:
  `(из типа)` или `(на поле)`.
- **Представление** — шаблон отображения/маска (категория D) и **тривиальная**
  видимость; `—` если нет.
- **Обяз.** — Да/Нет (обязательность значения).
- **Логика заполнения** — умолчание, источник списка, вычисление; `уточняется` для
  отложенной связи с функциями; `—` если поле заполняется только вручную.

Прочерки у простых полей — норма (плата за единую таблицу и однозначный парсинг).
Не переходи на под-строки `↳` — они ломают машиночитаемость (строка с пустыми
ячейками — конвенция, не структура).

## Примеры строк (все случаи)

| Поле | Тип | Формат | Представление | Обяз. | Логика заполнения |
|------|-----|--------|---------------|:-----:|-------------------|
| Дневной лимит | `Сумма.Лимит` | целое, 1–9 цифр, ≥ 0 *(из типа)* | разделитель разрядов при вводе | Да | — |
| Код валюты лимита | `Код.Валюта` | 3 заглавные буквы *(из типа)* | наименование рядом с кодом: `RUB — Российский рубль` | Да | список: {{platform:Справочник валют}}, только действующие |
| Счёт списания комиссии | `Счёт.Номер` | 20 цифр *(из типа)* | опция: `<номер счёта> — <валюта>, <остаток>, <тип счёта>`; значение — только номер | Да | список: счета клиента, валюта = валюте карты, не заблокированные |
| Индекс | — | 6 цифр *(на поле)* | — | Да | — |
| Комментарий | — | строка, до 500 символов *(на поле)* | скрыто, если тип счёта ≠ 2 *(видимость — тривиальная)* | Нет | по умолчанию: пусто |

Нетривиальная видимость в таблицу полей **не пишется** — она уходит строкой
`visibility` в FE-контроли; в «Представлении» можно оставить указатель
`видимость: см. SF-NN`.

## Пояснения под таблицей

Точечные пояснения — только к нетривиальным строкам (обогащённая опция выбора,
расходящиеся с атрибутом поля). Не дублируй под таблицей то, что уже сказано в
ячейках.
