# Единая таблица полей экранной формы

Все поля формы — в **одной** таблице. Порядок колонок фиксирован:

```
| Поле | Тип | Формат | Способ заполнения | Обяз. | Видимость | Логика установки значения |
```

> **Почему так (v2.1).** Прежняя колонка «Представление» была семантическим
> магнитом: в неё стекались и формат/пример, и логика вычисления значения.
> Разведено: **Формат** = как значение выглядит; **Видимость** = показ/скрытие
> **лейбла и значения**; **Логика установки значения** = откуда значение. Так три
> колонки резко различимы.

## Правила заполнения колонок

- **Поле** — имя поля как на форме.
- **Тип** — UI-тип поля из словаря (../../_shared/type-dictionary.md): Текст, Число,
  Дата, Список, Список с множественным выбором, Переключатель, Чекбокс, Блок
  загрузки, Гиперссылка, Заголовок и т.д.
- **Формат** — **ограничение И формат отображения** значения: тип-ограничение
  (длина, паттерн, набор литералов), маска ввода, шаблон опции/строки, пример,
  плейсхолдер, подсказка. Пометка: `(из типа)` — формат совпадает с атрибутом
  модели данных; `(на поле)` — расходится (адрес разбит на индекс/город/улицу
  против единого атрибута). **Сам атрибут-источник значения сюда не пишется** — он
  в «Логике установки значения». Помимо маски/примера: состав полей в
  **полуразвёрнутом** виде блока, статус/иконки в заголовке (когда элемент не
  скрыт видимостью). **Не** дублировать сюда «наименование не отображается» — это
  «Видимость».
- **Способ заполнения** — **{Вручную | Выбор из списка | Автоматически}**:
  - «Вручную / выбор из списка» — **«Выбор из списка»**.
  - **«Автоматически» = недоступность для ручного ввода.** Условную недоступность
    пиши здесь: «Автоматически, если заполнен Идентификатор физлица в ЕСК; иначе
    Вручную».
  - Автозначение с возможностью правки — это умолчание: способ «Вручную», значение
    — в «Логике установки значения».
- **Обяз.** — Да/Нет. Условная обязательность — `см. SF-NN`.
- **Видимость** — показ/скрытие **наименования (лейбла) и значения** поля: по
  умолчанию лейбл отображается; условие **Если … то … иначе …** без оговорки
  относится к **обоим**; «Наименование поля не отображается» — явная оговорка
  только про лейбл (см. categories.md, «Наименование и значение»). Также: «Всегда»
  / `см. SF-NN` / «Скрытое поле (не отображается)»; **режим блока** (свёрнут /
  развёрнут / полуразвёрнут) — references/categories.md. **Формул вычисления
  значения здесь быть не должно**.
- **Логика установки значения** — как поле **получает значение**: умолчание,
  источник списка (фильтр), **вычисляемая формула (присваивание: `=`)**, автозаполнение
  из атрибута сущности; `уточняется` — отложенная связь с функциями; `—` если поле
  заполняется только вручную и без умолчания.

Прочерки — норма. Без под-строк `↳`.

**Форматирование ячеек** — по ../../_shared/formatting.md: `<br>`-разделение смысловых
блоков, жирные «Если/то/иначе», отдельными блоками «**По умолчанию:**»,
«**Источник списка:**», «**Шаг N**». Не сливать текст в один поток.

**Сохранение при переписывании (v3.0):** перенос содержимого поля в ячейку — это
семантическое **дополнение, а не переписывание с потерей**. Сохраняй **как есть**
иконки `<img>`, ссылки (в виде markdown `[текст](url)`), списки (в виде `<br>- …`), выделения,
**вложенность условий** (отступ `&nbsp;`×4 на уровень) и токены `<Имя атрибута>`.
Нераспознанное — маркером `[НЕ РАСПОЗНАНО: …]`, не выбрасывать. Детали —
../../_shared/formatting.md.

## Правила-различители (куда что кладём)

- **Опция списка:** статическая склейка атрибутов (`<номер> — <валюта>, <остаток>`)
  — **Формат** (это вид). Условный выбор значения (`краткое наименование, если
  указано, иначе полное`) — **Логика установки значения** (есть условие — это
  вычисление значения).
- **Пример/маска/плейсхолдер/подсказка** — всегда **Формат**.
- **Атрибут-источник значения** (`= [Заявка].<Код типа счета>`) — **Логика
  установки значения**; в «Формате» остаётся только пометка `(из типа)`.
- **Заголовки/вычисляемые лейблы** (текст, собираемый из данных) — **Логика
  установки значения** с `=`.
- **Каскад «при изменении A очищается/пересчитывается B»** — **Логика установки
  значения** поля **B** (`значение сбрасывается/переформируется при изменении поля
  <A>`). Массовый сброс — строкой у заголовка блока. Это не контроль и не реакция.
- **Автопреобразование ввода** (верхний регистр, тримминг) — **Формат**.
- **Условная недоступность** («недоступно, если …») — **Способ заполнения**
  («Автоматически, если …; иначе Вручную»).

## Что в таблицу полей НЕ пишется

- **Реакции** (подсветка поля, показ/скрытие сообщения об ошибке, модалки-
  подтверждения, плашки) — не свойство поля — единый документ реакций сервиса. НО:
  каскадная очистка/пересчёт значения — «Логика установки значения» поля-получателя;
  недоступность — «Способ заполнения» (это не реакции).
- **Необходимость BE-близнеца** источника списка — не на форме; она фиксируется в
  FE-документе контролей (раздел «Кандидаты в data-model контроли»).
- **Нетривиальная видимость** и **relational-проверки** — в FE-контроли; в форме —
  указатель `см. SF-NN`.

## Примеры строк

| Поле | Тип | Формат | Способ заполнения | Обяз. | Видимость | Логика установки значения |
|------|-----|--------|-------------------|:-----:|-----------|---------------------------|
| Дневной лимит | Число | целое, 1–9 цифр, ≥ 0; разделитель разрядов при вводе *(из типа)* | Вручную | Да | Всегда | — |
| Валюта | Список | 3 заглавные буквы; формат отображения `<Наименование>`; пример `Российский рубль` *(из типа)* | Выбор из списка | Да | Всегда | Значение — [Заявка].<Код валюты>. Источник: справочник валют, только действующие. По умолчанию: единственная валюта |
| Наименование ЭФ | Заголовок | — | Автоматически | Нет | Всегда | Если статус == None, то = `"Заявка"`; иначе = `"Заявка" + " №" + <Номер>` |
| Комментарий | Текст | до 500 символов *(на поле)* | Вручную | Нет | Если тип счёта == 2, то отображается, иначе не отображается | По умолчанию: пусто |
| Список сотрудников | Список | … | Выбор из списка | Нет | Наименование поля не отображается. Отображается, **если** «Кому выпустить карту» == "Сотруднику" | … |
| Блок «Реквизиты ГОЗ» | Заголовок | — | Автоматически | — | `см. SF-04` | — |
| Блок «Анкета» | Заголовок | Заголовок анкеты: …; статус …; в полуразвёрнутом виде — поля: ФИО, … (см. «Действия») | Автоматически | — | Наименование блока не отображается. Контейнер блока — всегда; умолчание: развёрнут при первом открытии; режимы содержимого — см. categories.md | Каскады — см. categories.md |
| Мобильный телефон *(в блоке «Анкета»)* | Текст | маска `+7 (9ХXX) XXX-XX-XX` | Вручную | Да | В развёрнутом и полуразвёрнутом виде блока — отображается; в свёрнутом — не отображается (кнопка «Скрыть»/«Показать анкету», см. «Действия») | — |

«Наименование ЭФ» — вычисляемый лейбл — формула с `=` в «Логике установки
значения». «Валюта» — пример/маска в «Формате», источник и умолчание в «Логике».
«Комментарий» — тривиальная видимость (одно условие) для **лейбла и значения**.
«Список сотрудников» — лейбл скрыт явно, условие — только для значения.
«Блок ГОЗ» — видимость блока (крит. 2) вынесена в контроль. «Блок Анкета» —
наименование блока в «Видимости»; режимы свёрнутости — см. categories.md.

## Антипример: ошибочная классификация (поле «Организация»)

Типичная **механическая** ошибка при column mapping из Confluence:

| Поле | Тип | Формат | Способ заполнения | Обяз. | Видимость | Логика установки значения |
|------|-----|--------|-------------------|:-----:|-----------|---------------------------|
| Организация | Текст (поиск/выбор) | Отображается краткое наименование организации; если оно пусто — полное наименование | Выбор из списка | Да | Всегда | Источник списка: … |

**Ошибки:**

1. **Тип** — скопирован legacy «Текст» из Confluence вместо **Список** (фактическое
   поведение — выбор из списка организаций; см. type-dictionary.md, нормализация).
2. **Формат** — условное **вычисление значения** (`краткое … иначе полное`) вместо
   шаблона/маски/примера. По правилам-различителям выше это **Логика установки
   значения** (опция-выбор-по-условию), не «Формат».
3. Гибридный тип `Текст (поиск/выбор)` — вне словаря UI-типов.

**Правильно (смысловой эквивалент):**

| Поле | Тип | Формат | Способ заполнения | Обяз. | Видимость | Логика установки значения |
|------|-----|--------|-------------------|:-----:|-----------|---------------------------|
| Организация | Список | краткое наименование, если указано, иначе полное *(из типа)* | Автоматически, если …; **иначе** Выбор из списка | Да | Всегда | **Источник списка:** … **По умолчанию:** … = \<Краткое наименование\> **если** не пусто, **иначе** \<Полное наименование\> … |

Разбор — только через проход **2a** (инвентарь фрагментов по осям), не перенос
колонок Confluence один-к-одному.
