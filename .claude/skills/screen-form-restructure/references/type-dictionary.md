# Словарь типов полей (UI-типы ЭФ)

Колонка **«Тип»** в таблице полей заполняется **каноническим UI-типом виджета**
из словаря ниже — **не** копированием колонки Confluence «Тип поля» (см. раздел
«Нормализация типа»). Отдельный словарь типов данных/форматов значений в проекте
пока не ведётся; при необходимости расширим.

## Базовый набор (закрытый, расширяемый)

- **Текст** — однострочный/многострочный ввод.
- **Число** — числовой ввод.
- **Пароль** (Password) — маскируемый ввод.
- **Поиск** (Search) — ввод с контекстным поиском/подсказками.
- **Дата** (Date) — выбор даты (календарь).
- **Список** (Dropdown / Select) — выбор одного значения из списка.
- **Список с множественным выбором** (Multi-select) — выбор нескольких значений.
- **Переключатель** — набор взаимоисключающих значений (radio / toggle).
- **Чекбокс** — бинарный флаг.
- **Блок загрузки** — область загрузки файла (drag-and-drop).
- **Гиперссылка** — ссылка/кнопка-ссылка (в т.ч. Button-link, кнопка скачивания):
  ссылкой является **весь** элемент.
- **Текст со ссылками** — статический текст, внутри которого **отдельные
  фрагменты** являются гиперссылками (ядро — текст, ссылки — спаны внутри него).
  Перечень фрагментов-ссылок и действие каждого задаются атрибутом
  **«Действие»** правила поля («нажатие на "<фрагмент>" — <что открывает>») —
  тип и «Действие» вместе полностью определяют виджет для генерации кода.
  Не путать с «Гиперссылка»: там кликабелен весь элемент, здесь — только
  фрагменты.
- **Заголовок** — служебная строка-заголовок раздела/блока (не поле ввода).

## Приведение нестандартных типов исходника

Типы из выгрузки Confluence приводи к ближайшему каноническому, сохраняя смысл:

| В исходнике | Канонический тип |
|-------------|------------------|
| «Иконки/Текст», «Иконка статуса + Текст» | Заголовок (декоративный, если не поле ввода) |
| «Текст + гиперссылки», «Текст с ссылкой» | **развилка по нессылочному ядру**: ссылкой является весь элемент → Гиперссылка; ссылки — фрагменты внутри текста → **Текст со ссылками** (не «Текст»: тип потеряет интерактивность) |
| «Кнопка-ссылка на скачивание (Button-link)» | Гиперссылка |
| «Текст» при «Выбор из списка» / lookup / combobox в логике заполнения | **Список** |
| «Текст» при контекстном поиске без фиксированного списка опций | **Поиск** |
| «Текст» при однострочном/многострочном вводе без списка | Текст |
| «Иконка», «Изображение» (декоративный элемент, не ввод) | Заголовок или отдельный
  декоративный тип по смыслу; не «Текст» |
| «Тип сообщения: Информер» и аналоги | не колонка «Тип» поля ввода — отдельная
  строка **Информер** / реакция вне таблицы полей (см. categories.md) |

## Нормализация типа (обязательно)

Колонка Confluence **«Тип поля» — не источник истины** для колонки «Тип» SFR.

**Алгоритм:**

1. Определи **фактический виджет** по «Логике заполнения», «Комментарию», макету,
   действиям пользователя (выбор из списка, переключатель, загрузка файла, ввод текста…).
2. Приведи к **каноническому** UI-типу из базового набора выше.
3. Legacy «Текст» при «Выбор из списка организаций…» — **Список** (не «Текст
   (поиск/выбор)» и не гибрид вне словаря).
4. Контекстный поиск по справочнику **без** фиксированного enum опций — **Поиск**;
   выбор из заранее сформированного списка записей — **Список**.

**Запрещено:** изобретать гибридные типы вне словаря (`Текст (поиск/выбор)`,
`Список с автокомплитом` и т.п.) без **явного** добавления типа в этот файл.

**Детектор ошибки:** «Тип» = Текст (или гибрид), а «Способ заполнения» = Выбор из
списка — результат **механический**, требуется нормализация типа.

Если встречается **действительно новый** UI-тип, которого нет в наборе — добавь его
в этот список (не выдумывай разовую нотацию молча) и используй.

## Кандидаты в словарь форматов значений (опционально)

Повторяющиеся/общесистемные **форматы значений** (ИНН — 12 цифр, БИК — 9 цифр,
номер счёта — 20 цифр, код валюты — 3 буквы) можно собрать отдельным
списком-предложением на решение аналитика — как заготовку под будущий словарь
форматов. Это **не** колонка «Тип» (там UI-тип); формат значения живёт в колонке
«Формат». Словарь форматов самовольно не создаётся — только кандидаты.

```
| Кандидат формата | Ограничение | Поля-источники |
|------------------|-------------|----------------|
| Счёт.Номер | ровно 20 цифр | Номер счёта, Счёт списания комиссии |
| Код.Валюта | 3 заглавные буквы | Валюта |
```
