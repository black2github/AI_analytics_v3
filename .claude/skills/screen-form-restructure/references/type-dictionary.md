# Кандидаты в словарь типов

Скилл **не создаёт** словарь типов самовольно — он выделяет **кандидатов** на
решение аналитика.

## Как выделять

Кандидат в тип — формат, который:

- повторяется у нескольких полей этой формы или заведомо общесистемный (ИНН, БИК,
  номер счёта, ОГРН, код валюты, суммы/лимиты, даты);
- принадлежит полю, совпадающему с атрибутом модели данных (иначе тип общим не
  будет — см. границу «из типа / на поле»).

Одноразовые ограничения («до 500 символов» у комментария) в словарь не идут —
остаются `(на поле)`.

## Формат артефакта

Отдельный список в конце переформатированной ЭФ или отдельным файлом:

```
## Кандидаты в словарь типов (на решение аналитика)

| Кандидат | Ограничение | Поля-источники | Уже есть в словаре? |
|----------|-------------|----------------|---------------------|
| Счёт.Номер | строка, ровно 20 цифр | Счёт списания комиссии | уточнить |
| Код.Валюта | 3 заглавные буквы; из справочника валют | Код валюты лимита | уточнить |
```

Если словарь типов в проекте уже существует — сверь кандидатов с ним и ссылайся
на существующие типы вместо предложения новых.
