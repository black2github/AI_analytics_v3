# Чеклист финальной сверки

## Полнота
- [ ] Каждая характеристика исходника (формат, маска, видимость, умолчание,
      источник списка, подсказка, проверка) нашла свою колонку или строку
      контроля. Ничего не потеряно.
- [ ] Все поля исходника присутствуют в таблице полей.
- [ ] Встроенные в описание формы проверки извлечены в FE-контроли (C) либо в
      «Формат» (A) — не остались прозой в теле.

## Категории
- [ ] В «Представлении» нет ничего, что может провалиться или зависит от других
      полей (нетривиальная видимость — только в контролях типом `visibility`).
- [ ] Источник списка — в «Логике заполнения», не в контролях; необходимость
      бэкового близнеца помечена.
- [ ] Формат помечен источником: `(из типа)` только при совпадении поля с
      атрибутом; расходящиеся поля — `(на поле)`.
- [ ] Литеральный enum → Формат (A); lookup в справочник/сущность → контроль (C).
- [ ] Отложенная связь с функциями помечена `уточняется`, нотация не выдумана.

## Структура и текст
- [ ] Порядок колонок: Поле | Тип | Формат | Представление | Обяз. | Логика
      заполнения. Одна таблица, без под-строк `↳`.
- [ ] Условия/сообщения/описания — дословно, без усечений и «…».
- [ ] Ссылки исходника сохранены как markdown-ссылки.
- [ ] `visibility`-строки: сообщение об ошибке `—`.

## Имена и связи
- [ ] Имена файлов — из оригинальных минимальной правкой; `title`/`doc_id`
      согласованы; `related` двусторонний (ЭФ ↔ FE-контроли).
- [ ] FE-документ ссылается на экранную форму (не на функции).
- [ ] Кандидаты в типы — списком-предложением, словарь не создан самовольно.
- [ ] Выходные файлы — в каталоге оригинала.

## Сообщить пользователю
- [ ] Спорные классификации (граница тривиальной/нетривиальной видимости,
      кандидаты в типы, помеченные `уточняется`) — перечислить для согласования.
