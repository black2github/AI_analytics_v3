# Технические правила разбора (общие с control-split)

Правила идентичны control-split (references/edge-cases.md там); здесь — сжатая
выжимка, чтобы скилл был самодостаточен.

- **Треугольные скобки**: имена атрибутов/полей в нотации `<Имя>` защищай ДО
  снятия HTML-тегов (маркером «…»), исключая имена HTML-тегов (strong, p, ul, li,
  a, td, br…). Протягивай по `rowspan`.
- **Ссылки не терять**: `<a href>` заменяется на markdown `[текст](href)` первым шагом
  обработки ячейки. Относительные `.md` и Confluence-ссылки несут навигацию.
- **Таблицу разбирай как сетку** с раскрытием `rowspan`/`colspan` в матрицу
  фиксированной ширины — число ячеек в строках плавает.
- **Не обрезать текст**: условия, сообщения, описания — целиком, без `[:N]` и «…».
  В скриптах не оставлять отладочных срезов.
- **После разбора сверь** несколько строк с исходником глазами: колонки не
  съехали, имена полей не пусты, ссылки на месте.
