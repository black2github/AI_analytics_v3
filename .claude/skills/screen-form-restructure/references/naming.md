# Именование и размещение (общие с control-split)

- **Имя файла — из оригинального имени минимальной правкой**, без пробелов
  (дефисы), `.md`, URL-safe. Не пересобирать из «логического» названия.
- Переформатированная ЭФ: имя исходного файла ЭФ **сохраняется** (это та же
  страница в целевом виде). Если нужно держать обе версии рядом — суффикс
  согласуй с пользователем, по умолчанию замещение.
- FE-контроли: `FE-Контроли` после префикса в имени файла ЭФ:
  `[КК_ВК]-ЭФ-Клиента-страница-Анкеты.md` →
  `[КК_ВК]-FE-Контроли-ЭФ-Клиента-страница-Анкеты.md`.
- Артефакт кандидатов в типы: `<имя-ЭФ>-Кандидаты-в-словарь-типов.md` либо раздел
  в конце ЭФ (по объёму).
- **`title` и `doc_id` согласованы с именем файла** (та же основа; `title` —
  пробелы, `doc_id` — `{{<service>: <текст title>}}`). Рассинхрон ломает
  смарт-ссылки.
- **`related`** — строка со смарт-ссылками через запятую (не YAML-массив): ЭФ ↔
  FE-контроли двусторонне.
- Квадратные скобки в именах: артефакт выгрузки `_КК_ВК_` → канонично `[КК_ВК]`.
- **Каталог**: выходные файлы — рядом с обрабатываемым оригиналом.
