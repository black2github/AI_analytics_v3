# 🤖 Интеграция LLM-агента аналитика требований

## 📋 Обзор

Создан агент-аналитик требований с возможностями:
- ✅ Поиск требований по ключевым словам (RAG)
- ✅ Анализ страниц Confluence
- ✅ Проверка соответствия шаблонам
- ✅ Диалоговое взаимодействие с памятью

---

## 📦 Структура файлов

```
app/
├── agents/
│   ├── __init__.py                  # (создать пустой)
│   ├── agent_config.py              # Конфигурация агента
│   ├── requirements_agent.py        # Основной класс агента
│   └── agent_tools.py               # Инструменты агента
└── routes/
    └── agent.py                     # API endpoints
```

---

## 🔧 Шаги интеграции

### Шаг 1: Создание структуры

```bash
# Создайте директорию для агента
mkdir -p app/agents

# Создайте пустой __init__.py
touch app/agents/__init__.py
```

### Шаг 2: Добавление файлов

Скопируйте созданные файлы:
- `agent_config.py` → `app/agents/`
- `requirements_agent.py` → `app/agents/`
- `agent_tools.py` → `app/agents/`
- `agent.py` → `app/routes/`

### Шаг 3: Установка зависимостей

```bash
# Если ещё не установлены
pip install langchain langchain-openai langchain-community
```

### Шаг 4: Регистрация маршрутов

В главном файле приложения (например, `main.py`):

```python
from app.routes import agent

# Регистрируем роутер агента
app.include_router(agent.router, prefix="/api", tags=["Agent"])
```

### Шаг 5: Переменные окружения (опционально)

Добавьте в `.env`:

```bash
# Настройки агента (опционально, есть дефолтные значения)
AGENT_MODEL=gpt-4                    # или gpt-3.5-turbo
AGENT_TEMPERATURE=0.3                # температура для агента
AGENT_MAX_ITERATIONS=5               # макс. итераций агента
```

## 🚀 Использование через Postman

### 1. Чат с агентом

**Endpoint:** `POST /api/agent/chat`

**Body (JSON):**
```json
{
  "message": "Найди требования по СБП",
  "service_code": "SBP",
  "reset_history": false
}
```

**Примеры запросов:**

```json
// Поиск требований
{
  "message": "Найди все функциональные требования к авторизации"
}

// Анализ страницы
{
  "message": "Проанализируй страницу 274628758 и проверь её на соответствие шаблону"
}

// Общий вопрос
{
  "message": "Какие есть требования к системе быстрых платежей?"
}

// Проверка шаблона
{
  "message": "Проверь соответствие страницы 123456 шаблону функционального требования"
}
```

**Ответ:**
```json
{
  "response": "Найдено 5 требований по СБП:\n\n1. [Авторизация через СБП]...",
  "service_code": "SBP",
  "tools_used": [
    {
      "tool": "search_requirements",
      "input": {"query": "СБП", "service_code": "SBP"},
      "output_preview": "..."
    }
  ],
  "chat_history_length": 2,
  "error": null
}
```

### 2. Сброс истории

**Endpoint:** `POST /api/agent/reset`

**Body:** Не требуется

### 3. Информация об агенте

**Endpoint:** `GET /api/agent/info`

### 4. Список инструментов

**Endpoint:** `GET /api/agent/tools`

## 🔍 Отладка

### Проверка работоспособности

1. **Проверьте логи:**
```bash
# Агент логирует все действия с префиксом [RequirementsAgent]
tail -f app.log | grep "RequirementsAgent"
```

2. **Проверьте доступность инструментов:**
```bash
curl http://localhost:8000/api/agent/tools
```

3. **Проверьте статус агента:**
```bash
curl http://localhost:8000/api/agent/info
```

### Частые проблемы

**Проблема:** Агент не находит требования
- **Решение:** Проверьте, что векторное хранилище заполнено (`/load_pages`)
- **Решение:** Проверьте `service_code` в запросе

**Проблема:** Ошибка "No module named 'langchain'"
- **Решение:** `pip install langchain langchain-openai`

**Проблема:** Агент не использует инструменты
- **Решение:** Проверьте, что используется модель с поддержкой function calling (gpt-4, gpt-3.5-turbo)

```
[RequirementsAgent] Processing message: 'Найди требования по СБП'
[search_requirements_tool] <- query='СБП', service_code=SBP, top_k=5
[search_requirements_tool] -> Found 5 results
[RequirementsAgent] Response generated successfully, tools used: 1
```
