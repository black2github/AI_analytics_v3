# Чеклист финальной сверки

## Смысл (ВЫСШИЙ ПРИОРИТЕТ — проверять первым)
- [ ] Результат по каждому полю — **полный смысловой эквивалент** оригинала: каждое
      условие, ограничение и умолчание присутствует (переформулировка допустима,
      огрубление/потеря — нет). Проверка смысла человеком-читателем, не сравнение строк.
- [ ] **Размещение:** фрагменты с `целевая_строка ≠ источник` (проход 2a) **не** в
      ячейке поля-источника; каскады — у получателя или у заголовка блока.
- [ ] В ячейках полей-триггеров нет перечислений чужих полей, которые только
      сбрасываются/переформируются при изменении триггера.
- [ ] Многосмысловые фрагменты разложены полностью (напр. «значение по умолчанию» →
      и способ получения значения в «Логику», и блокировка = автоустановка в «Способ
      заполнения»).
- [ ] Нет «приблизительно»/обобщений, скрывающих условие (напр. «единственный вид»
      вместо «единственный вид счёта для выбранного карточного продукта»).
- [ ] Всё, чью эквивалентность подтвердить нельзя, помечено `[НЕ РАСПОЗНАНО: …]` с
      дословным исходным текстом (не сглажено, не выброшено).

## Полнота
- [ ] Режимы **свёрнутости/развёрнутости** (кнопки «Показать»/«Скрыть», полуразвёрнутый
      вид) **не потеряны**: что видно в каком режиме — в «Видимости»; как выглядит
      при показе и состав полуразвёрнутого вида — в «Формате»; кнопки — в «Действиях»
      (см. categories.md).
- [ ] «Наименование блока/поля не отображается» перенесено в **Видимость** (не в
      «Формат»); условия без оговорки относятся к лейблу **и** значению.
- [ ] Все поля исходника присутствуют в таблице полей.
- [ ] Встроенные в описание формы проверки извлечены в контроли (`rule`) либо в
      «Формат» — не остались прозой в теле.

## Форма (таблица полей v2.1)
- [ ] Порядок колонок: Поле | Тип | Формат | Способ заполнения | Обяз. | Видимость |
      Логика установки значения. Одна таблица, без под-строк `↳`.
- [ ] **Тип** — из словаря UI-типов (references/type-dictionary.md).
- [ ] **Формат** содержит ограничение и **отображение** (маска, шаблон опции,
      пример, плейсхолдер, подсказка); пометка `(из типа)`/`(на поле)`. Атрибута-
      источника значения здесь нет.
- [ ] **Способ заполнения** ∈ {Вручную | Выбор из списка | Автоматически};
      недоступность выражена здесь (в т.ч. условная), а не как реакция. Заблокировано
      (видно, не редактируется) → сюда; скрыто (не показано) → «Видимость».
- [ ] **Видимость** содержит только показ/скрытие **лейбла и значения** (Всегда /
      Если-то-иначе / см. SF-NN / Скрытое поле; «Наименование поля не отображается»
      — явная оговорка про лейбл). **Формул вычисления значения здесь нет**.
- [ ] **Логика установки значения** — умолчание/источник списка/формула (со знаком
      `=`)/автозаполнение из атрибута; отложенное помечено `уточняется`.
- [ ] Разделители применены: опция-склейка → Формат, опция-выбор-по-условию →
      Логика; атрибут-источник → Логика; вычисляемый заголовок → Логика с `=`.
- [ ] Каскад «при изменении A очищается/пересчитывается B» → «Логика установки
      значения» поля **B** (не контроль, не реакция); массовый сброс — у заголовка
      блока. **Не** у поля A. Автопреобразование ввода (верхний регистр) → Формат.
      Условная недоступность → Способ заполнения.
- [ ] При частичном SFR: проход 2a по **всей** исходной ЭФ; отложенные каскады —
      в «Пояснения под таблицей» с целевой строкой и № в исходнике.
- [ ] Смешанные фразы «эффект + модалка» расщеплены: смена значения → форма,
      модальное подтверждение → реакция (вне формы).
- [ ] На форме **нет реакций** и **нет пометок «BE-близнец»** (близнецы — в
      FE-контролях, реакции — в едином документе сервиса).

## Контроли (references/controls.md)
- [ ] У каждого контроля есть **Название проверки**, передающее суть/цель (не
      пересказ условия, не `SF-NN`).
- [ ] **Условие** — как **Если … то … иначе …**, дословно, без усечений.
- [ ] В условии **нет реакций** и **нет логики получения значения** поля.
- [ ] `Тип` ∈ {`rule`, `visibility`}; `behavior` не используется.
- [ ] `visibility`-строки: «Сообщение об ошибке» = `—`; исход — в «Условии»; «Поля»
      в нотации `Объект ← зависимости`.
- [ ] Режим `R` — отдельная колонка перед группами; чистая видимость — только `R`;
      валидации-гарантии — `R` + `V`; чисто событийные — `V`.
- [ ] Каждый событийный контроль привязан `V` к группе; группы объявлены над
      таблицей.

## Видимость: механический критерий
- [ ] В контроли (`visibility`) вынесена только видимость, прошедшая хотя бы один
      счётный критерий: ветвление (>1 «если»/вложенность) ∨ **нетривиальная**
      разделяемость (см. оговорку в controls.md) ∨ ссылается на внешнее состояние
      (справочник/атрибут сущности/полномочия). Тривиальное «**Если** поле == X,
      **то** видимо, **иначе** нет» у нескольких полей — **не** выносить.
- [ ] Счёт применён верно: «если A и B» — одно «если» (не ветвление); «внешнее
      состояние» — ссылка **условия** на справочник/сущность/полномочия, а не
      происхождение значения поля.
- [ ] Размерность экрана оставлена у поля (среда рендеринга); но условие «размер +
      атрибут сущности» вынесено в контроль (по критерию 3 из-за атрибута).

## Форматирование ячеек (references/formatting.md)
- [ ] Правила formatting.md применены **ко всем** целевым ячейкам: колонки таблицы
      полей ЭФ **и** колонка «Условие» в таблице FE-контролей (жирные
      **Если**/**то**/**иначе**, `<br>`, отступы вложенности).
- [ ] Присваивание — `=`, сравнение в условиях — `==` (не оставлять двусмысленный
      `=` из исходника в условиях).
- [ ] В ячейках сохранено смысловое разделение (`<br>` / `<br><br>`), нет
      слипшегося потока текста.
- [ ] «Если» / «то» / «иначе» (любой регистр) — жирным.
- [ ] Ярлыки `**По умолчанию:**`, `**Источник списка:**` (и подобные) — отдельным
      блоком, жирным, с переносами до и после.
- [ ] «Шаг 1.», «Шаг №2» … — с новой строки, жирным.

## Сохранение элементов при переписывании (v3.0)
- [ ] Все **иконки** `<img>` из описаний полей сохранены в целевых ячейках.
- [ ] Все **ссылки** сохранены (markdown `[текст](url)`), не превратились в голый
      текст.
- [ ] **Списки** `<ul>/<ol>` сохранены (markdown-пункты через `<br>- `), не склеены
      через запятые.
- [ ] **Вложенность условий** сохранена (отступ `&nbsp;`×4 на уровень), не сплющена.
- [ ] Токены `<Имя атрибута>` не потеряны.
- [ ] Нераспознанные фрагменты помечены `[НЕ РАСПОЗНАНО: …]`, не выброшены.
- [ ] Вынесенное условие в форме не дублируется — только `см. SF-NN`.
