Формат — как в control-split (frontmatter, врезка Зоны 3, «Назначение» со ссылкой

```markdown
## Контроли формы

`R` — реактивный режим (отрисовка + изменение полей из «Поля»); `V` — событийный
(в рамках группы). `visibility`: «Сообщение об ошибке» = «—», исход — в «Условии».
«Поля» для visibility: `Объект ← зависимости`.

| ID | Ориг. | Название проверки | Тип | Поля | Условие | Сообщение об ошибке | R | Гр.1 | Гр.7 |
|----|-------|-------------------|-----|------|---------|---------------------|:-:|:-:|:-:|
| SF-01 | 2 | Дневной лимит не выше месячного | rule | Дневной лимит, Месячный лимит | Если дневной лимит ≤ месячный, то проверка пройдена, иначе ошибка | Дневной лимит не может превышать месячный | R | V |  |
| SF-02 | 9 | Видимость блока «Реквизиты ГОЗ» | visibility | Блок «Реквизиты ГОЗ» ← Тип счёта | Если тип счёта == 2, то блок отображается, иначе не отображается | — | R |  |  |
```
