# Характеристики поля: оси и колонки (v2.1)

Ось — **природа характеристики**, не место записи в исходнике. Каждый фрагмент
описания поля попадает ровно в одну ось → в свою колонку.

| Ось | Что это | Колонка формы | Может провалиться? |
|-----|---------|---------------|:---:|
| **Формат** | ограничение **и** формат отображения значения | «Формат» | ограничение — да; отображение — нет |
| **Контроль** | relational-правило (поля/контекст/сущность) | FE-контроли (`rule`) | да |
| **Видимость** | показ / скрытие | «Видимость» (тривиальная) / контроль (нетривиальная) | нет |
| **Способ заполнения** | как поле заполняется | «Способ заполнения» | — |
| **Логика установки значения** | откуда у поля значение | «Логика установки значения» | — |

## Формат (ограничение + отображение)

Свойство самого значения: формат/длина/паттерн/набор литералов (**ограничение**),
а также маска ввода, шаблон опции/строки, пример, плейсхолдер, подсказка
(**отображение**). Всё это — как значение выглядит. Пометка `(из типа)` — совпадает
с атрибутом модели данных; `(на поле)` — расходится.

- Ограничение может провалиться → если проверяется, становится `rule`-контролем.
- Отображение провалиться не может.
- **Атрибут-источник значения в «Формат» не пишется** — только пометка `(из типа)`;
  сам источник → «Логика установки значения».

Ловушка: «значение из справочника» — **не** литеральный enum. Литеральный enum в
условии («М»/«Ж») → Формат; lookup во внешний справочник → контроль.

## Контроль (relational-правило)

Проверке нужен контекст за пределами значения: межполевое правило, сверка со
справочником/сущностью, статус, полномочия. → FE-контроли (`rule`). Полная модель,
режимы `R`/`V`, критерий вынесения видимости — **references/controls.md**.

## Видимость

Показ/скрытие **значения поля и его наименования (лейбла)**. **По умолчанию —
колонка «Видимость»** (тривиальная, как Если-то-иначе).
В контроли (`visibility`) выносится **только по механическому критерию** (ветвление ∨
разделяют несколько полей/блоков ∨ внешнее состояние — references/controls.md).
Размерность экрана — **среда рендеринга**, не внешнее состояние: остаётся у поля.
Формулы вычисления значения в «Видимость» не пишутся — они в «Логике установки
значения».

### Наименование (лейбл) и значение

Колонка «Видимость» **всегда** описывает оба аспекта, даже если в исходнике про
лейбл сказано отдельной фразой («наименование поля не отображается», «название
поля не отображается»).

| Ситуация | Как писать в «Видимости» |
|----------|-------------------------|
| Обычное поле (лейбл и значение вместе) | «Всегда» или **Если … то … иначе …** — **без оговорки** условие относится **и к наименованию, и к значению** (напр. «Организация»: «ООО Запаска») |
| Лейбл скрыт, значение видно | Явно: «Наименование поля не отображается» + условие видимости **значения** (или «Всегда» для значения) |
| Лейбл виден только при определённом размере экрана, значение — иначе | Разделить в одной ячейке: напр. «Наименование поля не отображается при XL/L/M (см. «Формат»)» + условие видимости значения |
| Поле полностью скрыто на ЭФ | «Скрытое поле (не отображается)» — не показываются **ни** лейбл, **ни** значение |

**Не переносить** «наименование не отображается» в «Формат» — это видимость лейбла,
не маска/оформление значения. В «Формате» остаётся только отсылка «название поля
не отображается при XL/L/M (см. Видимость)», если в исходнике условие разнесено
между колонками описания.

### Свёрнутость / развёрнутость (режимы показа блока и полей)

**Не отбрасывать.** Фразы вроде «свёрнутый/развёрнутый/полуразвёрнутый вид»,
«наименование блока (поля) не отображается», «при нажатии Показать/Скрыть»,
«поля анкеты зависят от …» — **обязательны** в результате. Это не «лишний UI» и
не переносится только в «Действия». Если не удаётся разложить однозначно —
`[НЕ РАСПОЗНАНО: <исходный текст дословно>]`, **не опускать**.

**Отличие от условной видимости.** Условная видимость задаётся **значениями полей
формы** («если „Кому выпустить карту“ == … — поле видно»). Свёрнутость — **режим
представления** уже доступного содержимого: пользователь переключает кнопкой
(типично иконка уголка вниз = «Скрыть», вверх = «Показать» — сворачивает/
разворачивает **часть полей** блока; отдельно может быть полуразвёрнутый режим —
кнопка «Показать все данные» / «Скрыть все данные»). Алгоритм нажатия, модалки и
проверки BE при нажатии — раздел **«Действия»**; в таблице полей фиксируется
**что** видно **в каком режиме**.

**Разведение по колонкам (оба измерения обязательны, если есть в исходнике):**

| Что описывает исходник | Колонка | Пример формулировки |
|------------------------|---------|---------------------|
| Скрыт ли элемент в свёрнутом/полуразвёрнутом режиме блока; умолчание «блок создан развёрнутым» | **Видимость** | «В развёрнутом виде блока — отображается; в свёрнутом — не отображается (режим переключает кнопка „Скрыть“/„Показать анкету“, см. „Действия“)» |
| Скрыто ли **наименование** поля/блока при видимом значении (или наоборот) | **Видимость** | «Наименование поля не отображается» + условие видимости значения; **если** условие одно — оно по умолчанию для лейбла **и** значения |
| Как выглядит заголовок, статус, иконки; **какой набор полей** в полуразвёрнутом виде | **Формат** | «В полуразвёрнутом виде блока отображаются: ФИО, Кодовое слово, … (полный список — см. „Действия“, кнопка „Показать все данные“)» |
| Зависимость **состава** полей от других полей (тип карты, кому выпускается) | **Видимость** полей (или `см. SF-NN`) | Тривиальное условие — у каждого поля; нетривиальное — контроль |

**Три оси — не путать:**

- **Скрыто условием** (данные формы) → «Видимость» (Если-то-иначе / `см. SF-NN`).
- **Скрыто режимом блока** (кнопка свернуть/развернуть) → «Видимость» + ссылка на
  кнопку в «Действия»; **не** выносить в контроль только потому, что управляется
  кнопкой.
- **Заблокировано для ввода** → «Способ заполнения» (`Автоматически`), не видимость.

**Строка заголовка блока** (напр. «Блок „Анкета“») — нормальное место для
умолчания режима («при первом открытии — развёрнут»), скрытия наименования блока
(«Видимость») и отсылки к кнопкам сворачивания; **не сжимать** эти фрагменты только
до каскадов в «Логике».

## Способ заполнения

{Вручную | Выбор из списка | Автоматически}. «Автоматически» = недоступность для
ручного ввода (в т.ч. условная: «Автоматически, если <условие>; иначе Вручную»).
Автозначение, доступное для правки, — умолчание (способ «Вручную», значение — в
«Логике установки значения»).

## Логика установки значения

**Откуда у поля значение**: умолчание (и логика расчёта), источник списка (какие
опции, включая фильтрацию), вычисляемая формула (присваивание со знаком `=`),
автозаполнение из атрибута сущности; `уточняется` — отложенная связь с функциями.
В условиях внутри ячеек сравнение — `==`, не `=` (см. references/formatting.md).

**Источник списка — не контроль** (ограничивает предлагаемое, провалиться не
может). Но фронтовая фильтрация обходима → нужен **BE-близнец** (data-model
контроль). Необходимость близнеца фиксируется **не на форме**, а в FE-документе
контролей.

**Межполевой эффект на значение** — «при изменении поля A значение поля B
сбрасывается / переформируется» — тоже «Логика установки значения», и пишется **у
поля B** (свойства поля читают у поля): `значение сбрасывается/переформируется при
изменении поля <A>`. Массовый сброс («при изменении «Организация» очищаются все
поля раздела») — одной строкой в «Логике установки значения» **заголовка блока**,
чтобы не дублировать на каждом поле. Это не контроль (ничего не проверяет, ошибки
нет) и не реакция (не про информирование об ошибке).

## Правила-различители (частые ошибки)

- **Опция списка:** статическая склейка (`<номер> — <валюта>`) → **Формат**;
  условный выбор значения (`… если …, иначе …`) → **Логика установки значения**.
- **Пример/маска/плейсхолдер/подсказка** → **Формат**.
- **Атрибут-источник** (`= [Заявка].<…>`) → **Логика установки значения**.
- **Вычисляемый заголовок/лейбл** → **Логика установки значения** с `=`.
- **Каскад при изменении другого поля** (очистка/пересчёт B при изменении A) →
  **Логика установки значения** поля B.
- **Автопреобразование ввода** (верхний регистр, тримминг) → **Формат**.
- **Условная недоступность** (недоступно, если …) → **Способ заполнения**
  («Автоматически, если …; иначе Вручную»).
- **Заблокировано ≠ скрыто:** заблокировано (поле **видно**, но не редактируется) →
  «Способ заполнения» (`Автоматически`); скрыто (не показано) → «Видимость». Разные оси.
- **Лейбл ≠ значение (видимость):** «наименование/название поля не отображается» →
  **Видимость** (не «Формат»); условие без оговорки — для лейбла **и** значения.
- **Свёрнуто ≠ скрыто условием:** свёрнутость/полуразвёрнутость блока (кнопка
  «Показать»/«Скрыть») → «Видимость» полей + оформление режима в «Формате»
  заголовка блока; см. раздел «Свёрнутость / развёрнутость» выше. Не отбрасывать
  как «не относящееся к полям».

Различитель «что делает фрагмент»: **проверяет → может провалиться** → контроль;
**показывает/скрывает** → видимость; **меняет значение** (своё или другого поля) →
Логика установки значения; **как выглядит** → Формат; **как заполняется** → Способ
заполнения.

## Реакции — вне спецификации формы

Подсветка поля, показ/скрытие сообщения об ошибке, модальные окна-подтверждения —
**реакции**: не контроль и не колонка формы (детали и адрес единого документа
сервиса — в references/controls.md). Не путать с тем, что реакцией **не** является:
- **очистка/пересчёт значения** другого поля (каскад) → «Логика установки значения»
  поля, чьё значение меняется;
- **недоступность для редактирования** → «Способ заполнения» («Автоматически»).

Если исходная фраза смешивает эффект и подтверждение («поле очищается **и**
показывается модалка»), расщепи: смена значения → форма, модалка → реакция (вне).

## Границы применимости

- «Из типа» — только при совпадении формата с атрибутом; иначе «на поле».
- Наполнение данными функций — отложенный вопрос (`уточняется`).
- Задвоение форматов фронт/бэк через общий тип — не предмет скилла.
