## 🔄 Поток обработки запроса

### Сценарий 1: Простой вопрос (без инструментов)

```
Пользователь → "Что ты умеешь?"
     │
     ▼
[API] POST /agent/chat
     │
     ▼
[Agent] RequirementsAgent.chat()
     │
     ├─> [LLM] Анализ запроса
     │   └─> Решение: ответ без инструментов
     │
     ▼
[Response] "Я могу искать требования, анализировать страницы..."
```

### Сценарий 2: Поиск требований (с использованием инструмента)

```
Пользователь → "Найди требования по СБП"
     │
     ▼
[API] POST /agent/chat
     │
     ▼
[Agent] RequirementsAgent.chat()
     │
     ├─> [LLM] Анализ запроса
     │   └─> Решение: использовать search_requirements
     │
     ├─> [Tool] search_requirements_tool()
     │   ├─> resolve_service_code("SBP")
     │   ├─> search_documents(query="СБП", service_code="SBP")
     │   └─> return JSON с результатами
     │
     ├─> [LLM] Формирование ответа на основе результатов
     │
     ▼
[Response] "Найдено 5 требований: 1. [Название]..."
```

### Сценарий 3: Многошаговый анализ (несколько инструментов)

```
Пользователь → "Найди требования по авторизации и проанализируй первое"
     │
     ▼
[Agent] Iteration 1
     │
     ├─> [Tool] search_requirements(query="авторизация")
     │   └─> Результат: page_id=123456, title="Авторизация"
     │
[Agent] Iteration 2
     │
     ├─> [Tool] analyze_page(page_id="123456")
     │   └─> Результат: детальный анализ
     │
     ├─> [LLM] Синтез финального ответа
     │
     ▼
[Response] "Найдена страница 'Авторизация'. Анализ показал..."
```

## 🧩 Компоненты системы

### 1. **API Layer** (`routes/agent.py`)
**Ответственность:** HTTP endpoints для взаимодействия с агентом

**Ключевые функции:**
- `agent_chat()` - основной endpoint для чата
- `agent_reset()` - сброс истории диалога
- `agent_info()` - информация о статусе
- `agent_tools()` - список доступных инструментов

**Особенности:**
- Асинхронные handlers с `anyio.to_thread.run_sync()`
- Pydantic модели для валидации
- Обработка ошибок с логированием

---

### 2. **Agent Core** (`agents/requirements_agent.py`)
**Ответственность:** Orchestration, reasoning, tool selection

**Класс:** `RequirementsAgent`

**Ключевые методы:**
- `__init__()` - инициализация LLM, tools, executor
- `chat()` - обработка сообщения пользователя
- `reset_conversation()` - очистка истории
- `get_conversation_summary()` - статистика сессии

**Компоненты:**
- **LLM** - языковая модель (OpenAI GPT-4/3.5 или локальная)
- **AgentExecutor** - LangChain orchestrator
- **Tools** - набор доступных инструментов
- **Chat History** - память диалога (in-memory)

**Принципы работы:**
1. Получает сообщение пользователя
2. LLM анализирует запрос и решает какие инструменты использовать
3. Выполняет инструменты последовательно
4. Синтезирует финальный ответ
5. Сохраняет историю

---

### 3. **Tools Layer** (`agents/agent_tools.py`)
**Ответственность:** Обёртки над существующей функциональностью

#### Tool 1: `search_requirements_tool`
```python
Вход: query, service_code, top_k
Выход: JSON с результатами поиска

Интеграция:
├─ app.rag_pipeline.search_documents()
└─ app.service_registry.resolve_service_code_by_user()
```

#### Tool 2: `analyze_page_tool`
```python
Вход: page_id, service_code, check_templates
Выход: JSON с анализом страницы

Интеграция:
└─ app.services.analysis_service.analyze_pages()
```

#### Tool 3: `check_template_compliance_tool`
```python
Вход: page_id, requirement_type, service_code
Выход: JSON с проверкой соответствия

Интеграция:
└─ app.services.analysis_service.analyze_with_templates()
```

**Особенности реализации:**
- Все инструменты возвращают JSON-строки (для LLM)
- Обработка ошибок с возвратом структурированных сообщений
- Логирование всех операций

---

### 4. **Configuration** (`agents/agent_config.py`)
**Ответственность:** Настройки агента и промпты

**Содержит:**
- `AGENT_SYSTEM_PROMPT` - системный промпт агента
- `AGENT_TOOLS_DESCRIPTIONS` - описания инструментов для LLM
- `AGENT_EXAMPLES` - примеры для few-shot learning
- Переменные окружения (model, temperature, max_iterations)

## 🔐 Безопасность и ограничения

### Текущие ограничения:
1. **Память:** In-memory (сбрасывается при перезапуске)
2. **Сессии:** Глобальный экземпляр агента (одна сессия на всё приложение)

## 📊 Производительность

### Типичные времена выполнения:

| Операция | Время | Комментарий |
|----------|-------|-------------|
| Простой ответ (без tools) | 1-3 сек | Зависит от LLM |
| Поиск требований | 2-5 сек | RAG + LLM response |
| Анализ страницы | 5-15 сек | Загрузка + анализ LLM |
| Проверка шаблона | 10-20 сек | Полный анализ |
| Многошаговый запрос | 10-30 сек | Несколько tool calls |

### Факторы влияющие на скорость:
- Выбор LLM (GPT-4 медленнее GPT-3.5)
- Размер контекста
- Количество инструментов в цепочке
- Кеширование (первый запрос медленнее)

## 🔧 Настройка и кастомизация

### Изменение модели LLM
```python
# В .env
AGENT_MODEL=gpt-3.5-turbo  # быстрее, дешевле
# или
AGENT_MODEL=gpt-4          # умнее, точнее
```

### Изменение системного промпта
Отредактируйте `AGENT_SYSTEM_PROMPT` в `agent_config.py`

### Добавление нового инструмента

1. Создайте функцию в `agent_tools.py`:
```python
def new_tool(param1: str) -> str:
    # Ваша логика
    return json.dumps(result)
```

2. Добавьте описание в `agent_config.py`:
```python
AGENT_TOOLS_DESCRIPTIONS["new_tool"] = {
    "name": "new_tool",
    "description": "...",
    "parameters": {...}
}
```

3. Зарегистрируйте в `requirements_agent.py`:
```python
Tool(
    name="new_tool",
    func=new_tool,
    description=AGENT_TOOLS_DESCRIPTIONS["new_tool"]["description"]
)
```
