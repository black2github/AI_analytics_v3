## 🎯 Возможности агента

### **Базовые функции** (MVP v1.0)

✅ **1. Поиск требований** (Tool: `search_requirements`)
```json
Запрос: "Найди требования по авторизации"
→ Использует RAG для поиска в векторном хранилище
→ Возвращает top-K релевантных требований с метаданными
```

✅ **2. Анализ страниц** (Tool: `analyze_page`)
```json
Запрос: "Проанализируй страницу 274628758"
→ Загружает страницу из кеша/Confluence
→ Выполняет LLM-анализ качества требований
→ Возвращает детальные рекомендации
```

✅ **3. Проверка шаблонов** (Tool: `check_template_compliance`)
```json
Запрос: "Проверь соответствие страницы 123456 шаблону FR"
→ Сравнивает структуру с эталонным шаблоном
→ Выявляет отклонения
→ Предлагает исправления
```

✅ **4. Диалоговое взаимодействие**
- Память о предыдущих сообщениях в рамках сессии
- Контекстные ответы на основе истории
- Автоматический выбор нужных инструментов

## 🚀 Быстрый старт (5 минут)

### **Шаг 1: Подготовка**
```bash
# Создайте директорию
mkdir -p app/agents
touch app/agents/__init__.py

# Скопируйте файлы из артефактов
# agent_config.py → app/agents/
# requirements_agent.py → app/agents/
# agent_tools.py → app/agents/
# agent.py → app/routes/
```

### **Шаг 2: Зависимости**
```bash
pip install langchain langchain-openai langchain-community
```

### **Шаг 3: Регистрация в main.py**
```python
from app.routes import agent
app.include_router(agent.router, prefix="/api")
```

### **Шаг 4: Запуск**
```bash
uvicorn app.main:app --reload
```

### **Шаг 5: Тест**
```bash
curl http://localhost:8000/api/agent/info
```

**Готово!** 🎉

## 📡 API Endpoints

### 1. **Чат с агентом**
```http
POST /api/agent/chat
Content-Type: application/json

{
  "message": "Найди требования по СБП",
  "service_code": "SBP",
  "reset_history": false
}
```

**Ответ:**
```json
{
  "response": "Найдено 5 требований...",
  "service_code": "SBP",
  "tools_used": [
    {
      "tool": "search_requirements",
      "input": {...},
      "output_preview": "..."
    }
  ],
  "chat_history_length": 2
}
```

### 2. **Сброс истории**
```http
POST /api/agent/reset
```

### 3. **Информация об агенте**
```http
GET /api/agent/info
```

### 4. **Список инструментов**
```http
GET /api/agent/tools
```

## 💡 Примеры использования

### **Пример 1: Простой поиск**
```json
{
  "message": "Найди все требования к авторизации"
}
```
→ Агент использует `search_requirements`
→ Возвращает список требований с кратким описанием

---

### **Пример 2: Глубокий анализ**
```json
{
  "message": "Найди требования по СБП и проанализируй самое важное детально"
}
```
→ Шаг 1: `search_requirements` - находит требования
→ Шаг 2: `analyze_page` - анализирует первое требование
→ Возвращает детальный отчёт

---

### **Пример 3: Проверка качества**
```json
{
  "message": "Проверь страницу 274628758 на соответствие шаблону функционального требования"
}
```
→ `check_template_compliance` с requirement_type="FR"
→ Возвращает отчёт с найденными проблемами

---

### **Пример 4: Мультишаговый диалог**
```
Запрос 1: "Найди требования по авторизации"
Ответ:   → Список из 3 страниц

Запрос 2: "Проанализируй первую"
Ответ:   → Детальный анализ (помнит контекст!)

Запрос 3: "Есть ли проблемы?"
Ответ:   → Конкретные проблемы и рекомендации
```

## 🔧 Конфигурация

### **Переменные окружения** (опционально)
```bash
# .env
AGENT_MODEL=gpt-4              # Модель для агента
AGENT_TEMPERATURE=0.3          # Температура (0-1)
AGENT_MAX_ITERATIONS=5         # Макс итераций
```

### **Дефолтные значения**
- Model: `gpt-4` (или из LLM_MODEL)
- Temperature: `0.3` (более детерминированный)
- Max iterations: `5` (макс шагов рассуждения)

## 🐛 Troubleshooting

### **Проблема: "No module named 'langchain'"**
**Решение:**
```bash
pip install langchain langchain-openai langchain-community
```

---

### **Проблема: Агент не находит требования**
**Решение:**
1. Проверьте что векторное хранилище заполнено:
```bash
curl http://localhost:8000/api/debug_collections
```

2. Загрузите тестовые данные:
```bash
curl -X POST http://localhost:8000/api/load_pages \
  -H "Content-Type: application/json" \
  -d '{"page_ids": ["274628758"], "service_code": "SBP"}'
```

---

### **Проблема: 500 Internal Server Error**
**Решение:**
1. Проверьте логи приложения
2. Убедитесь что OpenAI API key настроен
3. Проверьте статус агента: `GET /api/agent/info`
