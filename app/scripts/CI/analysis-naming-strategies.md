# Сравнение стратегий именования каталогов и файлов при миграции из Confluence

Сравниваются два подхода к формированию имён каталогов/файлов и структуры дерева:

1. **Текущий подход** — `app/scripts/migrate_confluence_tree.py` /
   `migrate_confluence_page.py`: без index-файла, имена коррелируют с содержимым
   (имя = заголовок Confluence как есть, кириллица), узел-с-детьми оформлен как
   «файл рядом с папкой», идентичность держит location-независимый
   `doc_id = {{SERVICE: title}}` (см. [design-smart-link-doc-id.md](design-smart-link-doc-id.md)).
2. **Чужой конвертер** — `app/scripts/converter.py`: `index.md` на каждую секцию,
   транслитерация имён в латиницу, идентичность/ссылки через `page_map` (путь
   выводится из имён).

## Ключевые различия (3 оси)

- **Чем заполняется имя:** заголовок как есть (кириллица) vs транслитерация в ASCII.
- **Узел-с-детьми:** `Заголовок.md` рядом с папкой `Заголовок/` vs `index.md` внутри
  папки `zagolovok/`.
- **Идентичность/ссылки:** location-независимый `doc_id` + реестр ID/title vs
  `page_map: id→relative_path`, где путь выведен из текущих имён.

## Таблица по критериям

| Критерий | Текущий подход (`migrate_confluence_tree.py`) | Чужой `converter.py` |
|---|---|---|
| Алфавит имён | Кириллица как есть: `safe_filename` режет только запрещённые в FS символы `\ / : * ? " < > \|`, пробел→`-`. Скобки `[КК_СК]` сохраняются | Транслитерация в латиницу (`TRANSLIT_DICT`), затем `[^\w\s-]`→удаляется, lowercase: `ef-klienta-...` |
| Длина имени | до 100 символов | до 80 символов |
| Узел с детьми | «файл рядом с папкой»: `Заголовок.md` + соседняя папка `Заголовок/` | «папка + index»: `zagolovok/` c `index.md` + дети внутри |
| Индекс-файл секции | нет — контент родителя в именованном `Заголовок.md` | `index.md` в каждой секции |
| Родитель без контента | только виртуальная папка `Заголовок/`, файл не пишется | всегда пишется `index.md` |
| Коллизии имён | `if filepath.exists(): skip` — дубликат **молча пропускается** | авто-суффикс `-1`, `-2` (`while md_path.exists()`) — потери нет |
| Идентичность / ссылки | `doc_id = {{SERVICE: title}}` — **независим от пути**; ссылки резолвятся через реестр ID+title → относительный путь (Pass 2) | `page_map: id→relative_path`, путь выведен **из имён**; резолв по id, затем title → отн. путь, иначе абсолютный URL |
| Папка вложений | общий `img/` рядом с `.md` | `<stem>_files/` на каждую страницу (только при наличии вложений) |
| Метаданные | YAML-frontmatter (тип, статус, doc_id, трассировка) + RAG/ChromaDB | нет — только `# Заголовок` и тело |
| Доп. артефакты | — | `structure_<root>.md` (оглавление), `export_log.txt` |

## Плюсы/минусы — текущий подход (кириллица, файл-рядом-с-папкой, без index)

**Плюсы**
- Имя файла = заголовок → узнаваемо, ищется `grep`/find по русскому слову, понятно в IDE.
- Нет «моря одинаковых `index.md`»: имя само идентифицирует контент.
- `doc_id` **не зависит от пути** → перенос каталогов не ломает ссылки (связь держит реестр).
- Путь зеркалит иерархию заголовков Confluence — человекочитаемая навигация.

**Минусы**
- Кириллица в путях: URL-кодирование (баг `%2B`/пробел), часть тулчейнов/SSG/CI/git-хостингов калечат не-ASCII.
- Длинные имена (100) + глубина → риск Windows `MAX_PATH` (260).
- `Заголовок.md` и `Заголовок/` с одинаковым стемом в одной папке — смешение файлов и папок на уровне.
- Коллизия заголовков под одним родителем → **тихая потеря** страницы (skip вместо суффикса).
- Нет `index.md` → SSG и GitHub не получают «лендинг секции».

## Плюсы/минусы — `converter.py` (транслитерация, `index.md`)

**Плюсы**
- ASCII-имена максимально портируемы: URL/CI/любая ОС/любой генератор статики; короче.
- `index.md` на секцию — идиоматично для SSG и GitHub (папка рендерит index/README как вход).
- Чёткая древовидность: узел-с-детьми = каталог; контент родителя самодостаточно внутри своей папки.
- Авто-дедуп `-1/-2` — страница не теряется при совпадении имён.
- `page_map` заранее → ссылки резолвятся одним проходом, с фолбэком по title.

**Минусы**
- Транслитерация **лоссова и неоднозначна**: не читается, не ищется по-русски; `ь/ъ` отбрасываются, `и/й` сливаются → больше коллизий и `-1`-суффиксов.
- `index.md` повсюду → в IDE десятки вкладок «index.md»; смысл несёт только (тоже транслит) имя папки.
- Идентичность привязана к пути/имени: переименование/перенос ломает `page_map`; стабильного location-независимого id нет.
- Смена заголовка → меняется слаг → ссылки «уплывают».
- Обрезка 80 + транслит склеивают разные длинные заголовки.

## Вывод — что под что

- **Текущий подход** — для **рабочей базы требований**: читаемость по-русски, поиск по
  заголовкам, стабильная идентичность при реорганизации, трассировка через frontmatter,
  RAG. Цена — риски не-ASCII путей, длины пути и тихих skip-коллизий.
- **`converter.py`** — как **экспортер под публикацию/портирование**: ASCII-безопасность,
  `index.md` для SSG, дедупликация, готовое оглавление. Цена — читаемость имён и привязка
  идентичности к пути.

## Гибрид

Оставить читаемые кириллические имена и `doc_id`-якорь, но добавить:

- (а) **обработка коллизий имён вместо тихого skip.** Решено упростить: при существующем
  файле — **перезаписывать с предупреждением**, а не дедупить по
  `confluence_page_id`/`doc_id`. Обоснование: миграция разовая, далее работа идёт по
  файловой системе под git, поэтому случайная перезапись безопасна и откатываема. Это
  снимает всю сложность различения «та же страница / другая страница» и дедупа пары
  `Заголовок.md` + `Заголовок/`. **Статус: РЕАЛИЗОВАНО.**
  - `save_page_file` (дерево) и `migrate_page` (одиночная): вместо `return` при
    `filepath.exists()` — `logger.warning("⚠ … ПЕРЕЗАПИСЫВАЕМ …")` и продолжение записи.
  - В дереве считается счётчик `stats["overwritten"]`, выводится в сводке миграции.
  - Реестры ссылок указывают на актуальный (перезаписанный) файл — «последний победил».
  - Тесты: `tests/test_migrate_overwrite.py`.
- (б) **опциональная генерация `index.md`/оглавления** для SSG — флаг `--with-index`,
  Pass 3 после резолва ссылок. **Статус: РЕАЛИЗОВАНО.**
  - `generate_section_indexes(base_dir)` кладёт навигационный `index.md` в каждую папку
    дерева: относительные ссылки на вложенные `.md` (заголовок из frontmatter) и подпапки.
  - Файл без frontmatter → линтер пропускает его по имени (`index.md`/`README.md`),
    индексатор — по отсутствию `service_code`; в требования/ChromaDB не попадает.
  - Цели ссылок обёрнуты в `<...>` (скобки/спецсимволы в кириллических именах не ломают ссылку).
  - Папки `img/` и сам `index.md` в перечень не входят; идемпотентно при повторном прогоне.
  - Тесты: `tests/test_section_indexes.py`.
