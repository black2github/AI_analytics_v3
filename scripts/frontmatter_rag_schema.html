Типы значений метаданных ChromaDB
ChromaDB принимает только str, int, float, bool. Списки не поддерживаются. Все поля с множественными значениями хранятся как строки через запятую.
Поля-списки → строка через запятую
jira_ids, reviewed_by, related, tags — хранятся как str.
Поиск через {"tags": {"$contains": "limits"}}.
В frontmatter YAML пишем строку: tags: "limits,card,daily" — не YAML-список!
Парсер frontmatter должен читать эти поля как есть, без преобразования в list.
Поле updated_date — только строка, не datetime
ChromaDB не поддерживает datetime. Для сортировки по дате используй формат ISO YYYY-MM-DD (str). Лексикографическое сравнение строк в этом формате корректно работает с $gte / $lte.
chunk_id и doc_id
В _create_multi_vector_chunked: chunk_id = f"{page_id}_chunk_{i}".
Поскольку page_id теперь содержит doc_id (corp-cards/limits/daily),
chunk_id станет corp-cards/limits/daily_chunk_0 — стабильный и читаемый идентификатор.
Поля, которые НЕ нужно хранить в метаданных
confluence_page_id нужен только в переходный период — через 6–12 месяцев после полной миграции можно убрать.
version как строка — только для фильтрации. Полная история версий — в git log, не в ChromaDB.
